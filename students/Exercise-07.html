What you need to know

HTML

  • There are lots of new concepts in this exercise. First we have a new tag <button> and </button> in lines 23-26. Whatever goes between the start and close tags is what is shown on the button. You can even use images insead of text and have a graphical button. Also note that clicking on a button does nothing unless there is some Javascript to tell the browser what to do when you click. The third button does nothing, however many times you click on it, because it has no listener.
  • Each button has a new attribute called id="". This is an identifier for the button. You use this id to refer to the button in your Javascript code. You can use whatever you like for the id, but it is a good idea to use something that describes what the button does. And be consistent when you name your buttons - it will make your Javascript code easier to read. The id must be unique in the document, so you can't have two buttons with the same id. You can have buttons with the same id on different web pages, but that is not always a good idea. Notice that in our example we included the name of the web page, exercise-07 in the id for each button to ensure the id won't clash with element ids on other web pages.
  • Also notice that the <img> tag in line 26 has a new attribute called title="". This is a tooltip that appears when you hover over the image. It is a good idea to use this attribute to provide a description of the image, especially if the image is not purely decorative.
  • Underneath the buttons on line 28 we have an empty <div> tag with an id of exercise-07-output. This is where the Javascript will write a message when the second and fourth buttons are clicked.

Javascript

  • We also have our first Javascript code. If some of this explaination sounds a bit like gobledegook, then check out the Advanced Witches and Wizards page for more detailed information about programming jargon. The javascript code is between the <script> and </script> tags in lines 29-82.
  • Note the use of comments in the Javascript code at lines 30, 35, 36 58-65 and elsewhere. Comments are used to explain what the code does and are ignored by the browser when it runs the code. They are very useful for you and anyone else who reads your code to understand what it does. You can add comments in Javascript using // for single line comments or /* ... */ for multi-line comments.
  • Each Javascript statement ends with a semicolon ;. This is not strictly necessary, but it is a good practice to include them. It makes your code easier to read and helps to avoid errors.
  • Javascript is case sensitive, so document and Document are not the same thing. You must use the correct case for all keywords, variables, and functions. Forgetting this is a source of many bugs - even for experienced web witches and wizards
  • Blocks of code are enclosed in curly braces { ... }. This is used to group statements together. For example, the code inside the addEventListener() method in lines 43-55 is a block of code that is executed when the button is clicked. Within that block are other blocks, such as the if statement in lines 50-52, and the else statement in lines 52-54.
  • You may remember from Exercise 1, each tag in the HTML document defines an element in the Document Object Model (DOM). So a <button> tag defines a button element, a <p> tag defines a paragraph element, and so on. As an aside, when you define styles in CSS, you are defining styles for elements. So when you say h1 { color: red; }, you are saying that all <h1> elements in the DOM should be red.
  • The first four lines of code (lines 30-33) in the <script> section defines a function called message(msg){}. This function takes one argument or parameter, msg, which is the message to be displayed. and a block of code that is executed to display the message when the function is called.
  • Functions are a fundamental building block of Javascript. They are reusable pieces of code that can be called with different arguments to perform a specific task. Each function has a name, in this case message, and can take zero or more arguments. In this case, the message() function is defined to display a message in a pop-up window when it is called. The function uses the built in Javascript alert() function to display the message in a pop-up window. It also adds the current time to the message using the new Date().toLocaleTimeString() method.
  • Another fundamental building block of Javascript is the variable. A variable is a named storage location for a value. In this case, we use the const keyword to define a constant variable, which means that the value of the variable cannot be changed. We use const for variables that we know will not change, such as the outputDiv variable in line 45. If we do try and change the value later, we will get an error. If we want to define a variable that can change, we use the let keyword instead of const. If we want to define a variable that is global and can be accessed from anywhere in the code, we use the var keyword. However, it is generally a good practice to use const or let instead of var as they have a more predictable scope and behavior.
  • An object is like a super variable that can hold multiple values and functions. We call the values properties and the functions methods. In our code we use the document object to access the elements in the HTML document. and the Date object to get the current time.
  • The document object is a built in object in Javascript that represents the HTML document, or more accurately, the DOM. It contains all the elements in the DOM, such as the buttons and the <div> elements. We use the document.getElementById() method to get a reference to an element in the document by its id at line 45. This method takes one argument, the id of the element you want to access. That is why we gave each button an id.
  • The Date object is a built in object in Javascript that represents a date and time. An instance of the Date object is created using the new Date() constructor in line 46 and the toLocaleTimeString() method is called on the Date object to get the current time as a string whioch is then stored in the (constant) variable timestamp. It is constant because we are not going to change its value later.
  • The timestamp is added to the message so that you can see when the message was displayed. In order to show the time on a new line in an alert box, we add the newline character "\n" in the string we pass to the alert() function in line 53.
  • Javascript has many built in functions and you put them together with the ones you write to make your programs.

Event Listeners

  • In our example we have four buttons and each button needs an event listener to "listen" for the user to click on that button and then do someting.
  • The first button has an id of exercise-07-button-1 and the second button has an id of exercise-07-button-2, etc. For each button we use the document.getElementById() method to get the button element. We then use the addEventListener() method to listen for a click event on the button. The addEventListener() method takes two arguments, the event to listen for (there are lots of events other than just a mouse click) and the function to call when the event occurs.
  • For the first button, in lines 30-33, we listen for a click event (and because the listener is attached to the button with id exercise-07-button-1, this only works when the user clicks on that button, not anywhere else)
    • When the click happens, we call the message() function with the message "Ouch! You clicked my button".
    • The message() function then pops up an alert box with the message it was passed and the timestamp of when the button was clicked.
    • Using a function to generate the alert message instead of simply calling the alert function directly is a good practice. It allows you to reuse the function, for example if you have multiple buttons that need to display a message with a timestamp when clicked. You can simply call the message() function with different messages for each button.
  • For the second button, we listen for a click event and trigger a function in lines 41-50. This is a bit more complicated than the button 1 example.
    • We start by creating const variables called outputDiv and timestamp in line 45-46, which hold a reference to the <div> element with the id of exercise-07-output and the current date and time object, respectively.
    • For the outputDiv variable, we do this using the document.getElementById() method. This allows us to access the <div> element and change its content when the button is clicked. If we want to change the id of the <div> element, we can do so in the HTML code, and in line 44, but we don't need to make any changes elsewhere in the Javascript code. Now each time we want to refer to the <div> element, we can use the outputDiv variable.
    • For the timestamp variable, we use the new Date() constructor to create a new Date object and then call the toLocaleTimeString() method on it to get the current time as a string. This is used to display the time when the button was clicked.
    • In line 45, we introduce another new property, innerHTML, which is a property of the <div> element (well every type of element has this property, but we mostly use it on divs). This property contains the HTML content of the element. We can use this property to check what is currently in the <div> element and change it.
    • Lines 50-54 contain an if statement. This is a very powerful feature of Javascript that allows you to make decisions in your code. The if statement checks if the innerHTML property of the <div> element contains the string "You clicked my button too!". If it does, then we know that the user has already clicked the button and we can assume that we have already written the message in the <div>. In this case, we set the innerHTML property to an empty string, which removes the message from the <div>. If the innerHTML property does not contain the string "You clicked my button too!", then we know that the user has not clicked the button yet, does not contain the message we set the innerHTML property to "You clicked my button too!" and add a timestamp of when the button was clicked.
    • Note that we can't check if the innerHTML property equals the string "You clicked my button too!" because it also has the timestamp in it so that would fail. Instead we use the includes() method to check if the innerHTML property contains the string "You clicked my button too!". This is a more flexible way to check if the message is already in the <div>.
    • So the second button is a toggle button. The first time you click it, it will write the message "You clicked my button too!" in the <div> element. The second time you click it, it will remove the message from the <div> element. This is a common pattern in user interfaces, where you want to toggle a message on and off when the user clicks a button.
  • Note that even though the third button has an id of exercise-07-button-3, it does not have an event listener, so it does nothing when you click it. There are some comments in lines 57-65, but these have no effect on the code. They are just there to explain why the button does nothing.
  • The fourth button does two things when you click it. First, it changes the image in the button from a witch to a wizard or vice versa, and second, it writes a message in the exercise-07-output <div> element. The code for this is in lines 69-81.
    • This time we have three const variables. The first is called img and is a reference to the <img> element inside the button we clicked on. this is a special property and refers to whatever element the listener is attached to. In this case, it refers to the button that was clicked. We use the querySelector() method to get the first <img> element inside the button.
    • The other variable is outputDiv and is the same as the one we used for the second button. It is a reference to the <div> element where we will write the message. Note that a variable only exists within the block of code it is defined in, so we can't use the outputDiv variable from the second button here. We have to define it again in this block of code. It is possible to define global variables that are accessible in all functions, but that is not a good practice as it is too easy to accidentally change them in the wrong place and cause bugs in your code.
    • In lines 74-80, we have another if statement. This time we check if the src property of the <img> element contains the string "witch" (the full property is src="/resources/images/young-witch.jpg"). If it does, then we know that the image is currently a witch and we change it to a wizard by setting the innerHTML property of the button to a new <img> element with the src set to the wizard image. We also write a message in the <div> element saying "Now I am a Wizard!".
    • If the src property does not contain the string "witch", then we know that the image is currently a wizard and we change it to a witch in the same way, and write a message in the <div> element saying "Now I am a Witch!".

Advanced Witches and Wizards

Events and listeners

Every time you do something in the browser, like moving the mouse, clicking a button, or typing in a text box, the browser sends an event to the DOM and you can create listeners attached to elements which listen for those events. When they hear an event they are listening for, they can do something. So you can use these events to trigger your Javascript code.

Other types of events include keydown and keyup for when the user presses or releases a key on the keyboard, mouseover and mouseout for when the mouse moves over or out of an element, and many more. You can find a full list of events in the MDN Web Docs.

Variables and constants

In Javascript, you can define variables using the let keyword, which allows you to change the value of the variable later, or the const keyword, which defines a constant that cannot be changed. It is a good practice to use const for variables that you know will not change, and let for variables that you know will change.

Variables are used to store values that you can use later in your code. You can think of a variable as a named box that holds a value. You can put a value in the box, take it out, and change it later. For example, you can define a variable called myVariable and assign it a value like this:

let myVariable = "Hello World";

Now you can use the variable myVariable in your code to refer to the value "Hello World". You can change the value of the variable later like this:

myVariable = "Goodbye World";

Now the variable myVariable holds the value "Goodbye World". You can also define a constant using the const keyword like this:

const myConstant = "This value cannot change";

Now you can use the constant myConstant in your code to refer to the value "This value cannot change". If you try to change the value of the constant later, you will get an error.

Variables and constants are very useful in programming because they allow you to store values that you can use later in your code. You can use them to store numbers, strings, objects, and other values. You can also use them to store references to elements in the DOM, like we did with the outputDiv variable in the example above.

Objects

In Javascript, an object is a collection of properties and methods. A property is a value that is associated with the object, and a method is a function that is associated with the object. You can think of an object as a box that holds multiple values and functions.

For example, the document object is an object that represents the HTML document. It has properties like title, which holds the title of the document, and methods like getElementById(), which allows you to get a reference to an element in the document by its id.

You can create your own objects in Javascript using the {} syntax. For example, you can create an object called myObject like this:

let myObject = {
    name: "My Object",
    value: 42,
    sayHello: function() {
        alert("Hello from " + this.name);
    }
};

Now you can use the object myObject in your code to refer to the object. You can access the properties of the object using the dot notation, like this:

alert(myObject.name); // "My Object"

You can also call the method of the object using the dot notation, like this:

myObject.sayHello(); // "Hello from My Object"

Objects are very useful in programming because they allow you to group related values and functions together. You can use them to represent real-world objects, like a person, a car, or a book, and you can use them to store data that you can use later in your code.

Functions and methods

A function is a reusable piece of code that can be called with different arguments to perform a specific task. You can think of a function as a recipe that you can use to cook something. You can call the function with different ingredients (arguments) to get different results.

For example, you can define a function called addNumbers(a, b) that takes two arguments and returns their sum like this:

function addNumbers(a, b) {
    return a + b;
}

Now you can call the function with different arguments to get different results:

alert(addNumbers(2, 3)); // 5
alert(addNumbers(10, 20)); // 30

In this example, the function addNumbers(a, b) takes two arguments, a and b, and returns their sum. You can call the function with different values for a and b to get different results.

A method is a function that is associated with an object. You can think of a method as a function that belongs to an object. For example, the sayHello() method in the myObject object we defined earlier is a method that belongs to the object. You can call the method using the dot notation, like this:

myObject.sayHello(); // "Hello from My Object"

Using the Developer Tools

If you are using Firefox as your browser you can use the Developer Tools to look at the JavaScript console. To do this, right click on the page and select Inspect (Q). This will open a new window at the bottom of the page where you can view the console in the console tab. If you have made any mistakes in your javascript that stop the program from running, you will see an error message in the console. These are known as bugs or syntax errors. There are other kinds of bugs, in particular semantic errors are where the program runs but does not do what you expect it to do. Those are often much harder to find and fix.

You can have your Javascript code write messages to the console using the console.log() method. For example, you can add a line of code like this in your Javascript code:

console.log("This is a message to the console");

This will write the message "This is a message to the console" to the console when the code is executed. You can use this to debug your code and see what is happening at different points in your code. It is a very useful tool for finding bugs in your code.

In the console you can also see any messages that are written to the console using the console.log() method.

In the console you can also type in JavaScript commands and see what they do. For example, you can type in alert("Hello World") and press enter to see what happens. You can also type in document.getElementById("exercise-07-output").innerHTML to see what the innerHTML of the <div> is. This is a great way to test your code and see what it does.
