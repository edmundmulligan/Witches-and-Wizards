Welcome to Exercise 6

In the last exercise you discovered styles. The styles in a document can get quite long so sometimes we put the styles into a seperate file, called a stylesheet, and link to it from the HTML document. This is a good idea because it makes the HTML document easier to read and edit. It also means that you can use the same stylesheet for multiple HTML documents. So you can change the styles in one place and all the documents will be updated. You can also quickly change the look of a whole website by changing the stylesheet.

You can change the style of just a bit of text by incorporaing it within a <span> block. You can also do things like make text larger or smaller with suitable CSS classes.

You can also change the style of a whole block of text by using a <div> tag. This is useful if you want to change the style of a whole section of your web page.

What you need to know

  • In this exercise the style in the <head> section from the previous exercise is now in a separate file called a stylesheet.
  • The stylesheet is linked to the HTML document using the <link> tag on line 12 in the <head> section.
  • The rel="stylesheet" attribute specifies that the link being loaded is a stylesheet.
  • The type="text/css" attribute specifies the contents of the stylesheet is css code.
  • The href="/resources/styles/styles-1.css" attribute specifies the location and name of the stylesheet.
  • You can change the look of your page by changing the file name in the href attribute to /resources/styles/styles-2.css
  • If you change the href attribute to a non-existant file such as /resources/styles/styles-3.css, the page will still work but none of the styling in your document will work.

Exercise

In this exercise you will create a HTML document and a stylesheet. You can use everything you have learned so far.

Copy the file template.html and name it exercise-06.html. If you prefer, you can create a new, empty file called exercise-06.html and copy the code from the template file into it.

Copy the file template.css and name it exercise-06.css. If you prefer, you can create a new, empty file called exercise-06.css and copy the code from the template file into it.

Add <link rel="stylesheet" type="text/css" href="/exercise-06.css"> to the head section of your HTML document. This will link your HTML document to your stylesheet.

Write some content in your HTML document. You can use the content from previous exercises or you can write your own new content. Make sure you use the class attribute to apply styles to your HTML elements. It doesn't matter if you haven't put the style elements in your stylesheet yet. You can add them later. Use some of the styles defined in exercise 5 so that when you change stylesheet later you will see the difference.

Now edit your style sheet to add some styles. Make sure you define the styles you used when you created the content for your HTML document. Don't forget, if you make a mistake in the stylesheet you will find that your html page does not display correctly. Tracking down these sorts of mistakes can be quite frustrating but is what real programmers have to do all the time. It is called debugging your code and you will learn a lot more about HTML and CSS if you can find and fix bugs in your code than if it always works perfectly first time. Note that you do not need to use <style> and </style> in a stylesheet. Your browser already knows the shhet contains CSS code rather than HTML because of the type="text/css" in the link attributes that load the stylesheet.

When you are finished save your document then in your browser go to http://localhost:8000/exercise-06.html to see what it looks like. Every time you make a change to a file, you need to save it and refresh the page in your browser to see the changes.

Edit the href attribute in the <link> tag to point to "/resourses/styles/style-1.css" and refresh the page. You should see the page change to the style defined in the style-1.css file.

Now edit the href attribute in the <link> tag to point to "/resourses/styles/style-2.css" and refresh the page to see a different style.

Advanced Witches and Wizards

When you browse the web, quite a lot happens in the background. It starts with you typing a URL into your browser's address bar or click on a hyperlink on a web page. The browser then sends a request to the server that hosts the website. The server processes the request and sends back the HTML, CSS, and JavaScript files that make up the web page. The browser then renders the page, applying styles from the CSS and executing any JavaScript code. This process happens very quickly, often in less than a second.

One of the ways that is used to make the pages load faster is to use a technique called caching. Caching is when the browser stores a copy of the web page or one of its files in its memory so that it doesn't have to download it again the next time you visit the page. This can make the page load much faster, especially if you visit the same page multiple times. For example, if you have serveral pages that all use the same CSS stylesheet, it only needs to be downladed for the first page you view. After that it is already downloaded so can render the page faster.

But there is a problem with caching. If the page changes, the browser might still use the old cached version instead of the new one. This also applies to stylesheets. If you make changes to your CSS file, the browser might still use the cached version of the stylesheet. This can lead to confusion because you might not see the changes you made to the styles. To avoid this, you can force the browser to reload the page and ignore the cached version by pressing Ctrl + F5 on Windows or Cmd + Shift + R on Mac.

You may well need to do this if you make a change to your CSS stylesheet and you don't see the change reflected in your web page. If you are using Firefox as your browser, you can also use the Developer Tools to disable caching while you are developing your page. To do this, right click on the page and select Inspect (Q) to open the Developer Tools, go to the Network tab, and check the "Disable Cache" option which is located towards the right of the panel. This will ensure that the browser always loads the latest version of the page and its files.

If you are using Firefox as your browser you can also use the Developer Tools to inspect your styles. To do this, right click on the page and select Inspect (Q) then select the Style Editor tab. You can even edit the stlyes in this window and see the effect of the original page. The changes you make here are not permenant, though. To make them permenant you will need to edit them in the stylesheet.
