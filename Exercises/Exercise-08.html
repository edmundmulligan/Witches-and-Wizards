Welcome to Exercise 08

This exercise is a bit easier than the last one. In the last exercise you wrote some Javascript and put it in the HTML web page. This makes the page a bit cluttered as it contains both your content (the HTML) and your program (the Javascript).

In this exercise we will move the Javascript into a separate file, just like we did with the CSS in Exercise 6. This will make your HTML file easier to read as it will only contain the content of the page. It will also make it easier to reuse your Javascript code in other pages.

The HTML part of the page (or at least the buttons) is the same as in the last exercise, but the Javascript is stored in a separate file. You will also add a <script> tag in the header of the HTML file to load the Javascript file.

You will create three buttons. The first button will pop up a message when you click it. The second button will turn a message in a <div> on and off when you click it. The third button has no listener so will do nothing when you click it.

Javascript

function message(msg) {
    alert(msg);
}
document.getElementById("exercise-08-button").addEventListener("click", function() {
    message("Ouch! You clicked my button");
});
document.getElementById("exercise-08-button-2").addEventListener("click", function() {
    if (document.getElementById("exercise-08-output").innerHTML == "") {
        document.getElementById("exercise-08-output").innerHTML = "You clicked my button too!";
    } else {
        document.getElementById("exercise-08-output").innerHTML = "";
    }
});

What you need to know

  • We are using the <script></script> block in the header section to load the Javascript code. Notice that (unlike in the last exercise where the javascript program was inside the block) there is nothing in the block itself and we are using the src="exercise-08.js" attribute to load the program.
  • The <script> tag also uses the defer attribute. This forces your broswer to wait until the whole HTML page has loaded before running any Javascript. Because the page is so small you probably won't notice the difference, but it is a good idea to use this attribute because the Javascript code uses HTML elements in the page (like the buttons and the output <div>) and if the Javascript code runs before the page has loaded, it won't be able to find the elements and will throw an error.
  • The Javascript is the same as in the last exercise (except the buttons have new names) but notice you don't need <script> and </script> tags in the javascript file because the broswer already knows this is Javascript by the ime you load it.
  • The actual code works in the same way as in exercise-07, so refere back to that if you need a reminder on how it works.

Exercise 08

In this exercise you will create a HTML document and a Javascript file to do something when you click the buttons. You will use the <script> tag with the src="" and defer attibutes in the head rather than the body of the document.

Copy the file template.html and name it exercise-08.html. If you prefer, you can create a new, empty file called exercise-08.html and copy the code from the template file into it.

Edit the file and add some content. You need to include the <button> and </button> tags, and the <div> and </div> tags, but do not put <script> and </script> tags in the body. Make sure you give each button and the <div> an unique id. You can use whatever you like for the id, but you will need to use the same ids in your Javascript code.

Add the <script> tag in the head of the document. Use the src="exercise-08.js" attribute to load the Javascript file and the defer attribute to make sure the Javascript code runs after the HTML page has loaded.

Copy the file template.js and name it exercise-08.js. If you prefer, you can create a new, empty file called exercise-08.js and copy the code from the template file into it.

Write some Javascript code to do something when you click the buttons. You can use the code from the example above as a starting point. You can use the document.getElementById() method to get the button elements and the addEventListener() method to listen for a click event on the buttons. You can use the alert() method to display a message when you click the first button. You can use the innerHTML property of the <div> element to display a message when you click the second button.

When you are finished save your document then in your browser go to http://localhost:8000/exercise-08.html to see what it looks like. Every time you make a change to a file, you need to save it and refresh the page in your browser to see the changes.

If you are using Firefox as your browser you can use the Developer Tools to look at the JavaScript console. To do this, right click on the page and select Inspect (Q). This will open a new window at the bottom of the page where you can view the console in the console tab. If you have made any mistakes in your javascript that stop the program from running, you will see an error message in the console. These are known as bugs or syntax errors. There are other kinds of bugs, in particular semantic errors are where the program runs but does not do what you expect it to do. Those are often much harder to find and fix.
