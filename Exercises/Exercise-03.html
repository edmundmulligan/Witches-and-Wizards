What you need to know

Tables

  • Tables and hyperlinks are more complicated than the tags we have already used, but they follow the same rules about needing closing tags that match the opening tags.
  • The table is defined by the <table> tag in line 21 and </table> tag in line 65. Everything between these tags is part of the table.
  • Note the indentations in this code. Indentation is not necessary - your web page will display in the broswer just fine without using indentations but if you use indentations it makes the code easier to read and understand. The biggest advantage is that consistent indentation helps you spot mismatched opening and closing tags.
  • Indentation is not part of the HTML standard, but it is a good practice to use it. You can use spaces or tabs for indentation, but be consistent.
  • A table is a grid of rows and colums - within the <table> and </table> tags you define the structure of the table.
  • In this example we are using the border="1" attribute in the <table> attribute to draw an outline of the table. This is not normally used any more - you can draw borders around tables in much more flexible ways using CSS which we will look at in later lessons.
  • The <caption> tag in line 22 is optional and is used to add a title to the table. The caption is displayed above the table in the browser.
  • After the <caption> tag we have three blocks <thead> (lines 23-28), <tbody> (lines 29-56), and <tfoot> (lines 57-64). These are used to define the header, body, and footer of the table. The header is displayed at the top of the table, the body is displayed in the middle, and the footer is displayed at the bottom. These blocks are optional, but they are useful for structuring the table and making it easier to read. If you omit them, the table will still work and the browser will treat everything as the body.
  • Within each block we have rows, which are defined by the <tr> and </tr> tags. Each row can contain one or more cells, which are defined by the <td> and </td> tags.. Rows in the header usually use the <th> and </th> tags instead of the <td> and </td> tags. The <th> tag is used to define a header cell, which is displayed in bold.
  • Within the individual <td> cells you can put whatever you like: text, images or lists. Your browser will work out the sizes of the cells and rows and columns for you. It is imprtant that you put the same number of cells in each row - if you don't the browser will still display the table but it will look funny.
  • In lines 59-63 there is a row that spans two columns which we do using the colspan="2" attribute in the first cell of the row. This allows you to merge cells in your table. If your table has more than two columns, you don't have to merge all the cells in the row, but you must make sure that the total number of columns in the row, including, the number of colspans matches the number of columns in the other rows. In this case it works out fine as the footer of the table is a hyperlink (see below) which doesn't apply to only one colums.

Hyperlinks

  • In lines 60 and 61 we have a hyperlink. This is defined by the <a> and </a> tags. Everything between these tags will be displayed as a clickable link. Note that the <a> tag is split over two lines, but this is just for readability. You can put the <a> tag on one line if you prefer.
  • You don't have to put text between the <a> and </a> tags - you could have an <img> instead. If you do use text, the link will be underlined and blue (although you can change that using CSS). Also when you hover the mouse over a link, whether it is text or an image, the pointer will change so you know the link can be clicked on.
  • You must use the href="" attribute in the <a> tag to specify the URL of the link , see the Advanced Witches and Wizards section below for more about URLs.
  • You can also use the target="_blank" attribute to open the link in a new tab or window. If you miss out the target="_blank" attribute the link will default to target="_self" which will open the link in the same tab or window and will replace the page you are on.

Exercise

In this exercise you will create two HTML documents, one containing a table and one joined with a hyperlink.

You will use the <table>, <thead>, <btody>, <tfoot>, <tr>, <th>, <td>, and <a> tags.

Copy the file template.html and name it exercise-03a.html. If you prefer, you can create a new, empty file called exercise-03a.html and copy the code from the template file into it. Then do the same to create a second file called exercise-03b.html

In the body of the first document add a table with a caption. Use the example in this page as a guide. You can use the same images and text as the example, or you can use your own images and text. You can also use the same table tructure as the example, or you can use your own structure.

In the second document, put any content you like. Use the previous exercises for ideas and inspiration.

In the first document, add a hyperlink to the second document. Use the <a href="exercise-03b.html"> tag to link to the second document. And in the second document, add a hyperlink back to the first document using the <a href="exercise-03a.html"> tag.

Make sure that the links work by opening the first document in your browser and clicking on the link to the second document. You should be able to navigate back and forth between the two documents.

Finally, in the first document, add a hyperlink to an external website. You can use any website you like, but make sure it is a valid URL. You can use the https://www.wikipedia.org/ website as an example. You will need to use the full URL and you want to make sure that the link opens in a new tab or window, so you will need to use the target="_blank" attribute. Your external link will look something like:

<a href="https://www.wikipedia.org/" target="_blank">Visit Wikipedia</a>

When you are finished save your document then in your browser go to http://localhost:8000/exercise-03a.html to see what it looks like. Every time you make a change to a file, you need to save it and refresh the page in your browser to see the changes.

Advanced Witches and Wizards

Tables

Many older websites use tables to layout the elements on a page page because they didn't have CSS available. This is not a good practice and is not recommended these days. This is mainly because tables do not scale well on different screen sizes, such as on mobile phones or tablets. It is much better to use CSS to layout the page, as you will discover in later lessons. However, tables are still used for displaying data, such as in spreadsheets or databases.

Hyperlinks

Hyperlinks are a fundamental part of the web. They allow you to link to other pages, images, or files. You can also use hyperlinks to link to other websites or to link to specific sections of a page.

The URL, or Uniform Resource Locator, is the address of a resource on the web. It is made up of several parts, including the protocol (such as http or https), the domain name (such as example.com), the port, the path to the resource (such as /path/to/resource), and any optional parameters known as a query string. A full URL is of the form protocol://server:port/resource?queryString.

So far you have only used the http protocol, which stand for Hyper Text Transfer Protocol. This was completly normal in the early days of the web, but all the data transmitted over the web using this protocol is unencrypted. This means that anyone, and by that I mean hackers, can intercept your data and read it. This is a security risk, especially if you are transmitting sensitive information, such as passwords or financial data. To address this issue, a new protocol was introduced called https, which stands for Hyper Text Transfer Protocol Secure. This protocol encrypts the data transmitted over the web, making it much more secure. You can tell if a website is using https by looking for the padlock icon in the address bar of your browser. For these exercises it is safe to use http because the server is running on your own computer so you are not transmitting any data over the internet. But if you link to external websites, you should always use https to ensure that the data is transmitted securely.

There are many other protcols you can use to link to resources on the web, such as ftp (File Transfer Protocol) for transferring files, and telnet for remote access to servers. We will not cover those protocols in this course, but you can find out more about them on the web. One that you may want to use is the mailto protocol, which allows you to create a link that opens the user's email client with a new message addressed to a specific email address. You can even include a query string to set the subject line of the email. This is useful for creating links that allow users to contact you directly from your website.

For example, to send Edmund feedback, you can use the following code:

<a href="mailto:edmund@edmundmulligan.name?subject=Feedback%20About%20The%20Web%20Witchcraft%20And%20Wizardry%20Exercises">send Edmund feedback</a>

This will create a link that opens the user's email client with a new message addressed to me, with the subject line set to "Feedback About The Web Witchcraft And Wizardry Exercises". The ?subject= part of the URL is called a query string, and it allows you to pass additional information to the email client or any other resource. Note that you can't put any spaces in the href, because your broweser will think that the URL has ended at the space. Instead, you can use a %20 code to represent a space in a URL. This is called URL encoding, and it is used to ensure that URLs are valid and can be transmitted over the web. You can also use other characters in URLs, such as %3A for a colon, %2F for a forward slash, and so on. You can find a full list of URL encoding characters here. In your own web pages, please use your own email address rather than mine!

Most web sites run on port number 80, which is the default port for the http protocol, or 443, which is the default port for https. In those cases you do not need to specify the port number in the URL. When you set up these exercises, you used python to start a simple web server on your own computer (which is known as localhost). Because python doesn't know if your computer is already running a web server, it doesn't use the default port number of 80 and uses port 8000 instead. This is why you access the exercises using the URL http://localhost:8000/.
