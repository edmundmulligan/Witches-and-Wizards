What you need to know

Tables

  • Tables and hyperlinks are more complicated than the tags we have already used, but they follow the same rules about needing closing tags that match the opening tags.
  • Note the indentations in this code. Indentation is not necessary - your web page will display in the broswer just fine without using indentations but if you use indentations it makes the code easier to read and understand. The biggest advantage is that consistent indentation helps you spot mismatched opening and closing tags.
  • Indentation is not part of the HTML standard, but it is a good practice to use it. You can use spaces or tabs for indentation, but be consistent.
  • A table is a grid of rows and colums - within the <table> and </table> tags you define the structure of the table.
  • In this example we are using the border="1" attribute in the <table> attribute to draw an outline of the table. This is not normally used any more - you can draw borders around tables in much more flexible ways using CSS which we will look at in later lessons.
  • The <caption> tag is optional and is used to add a title to the table. The caption is displayed above the table.
  • After the <caption> tag we have three blocks <thead>, <tbody>, and <tfoot>. These are used to define the header, body, and footer of the table. The header is displayed at the top of the table, the body is displayed in the middle, and the footer is displayed at the bottom. These blocks are optional, but they are useful for structuring the table and making it easier to read. If you omit them, the table will still work and the browser will treat the first row as the header, the last row as the footer, and the rest as the body.
  • Within each block we have rows, which are defined by the <tr> and </tr> tags. Each row can contain one or more cells, which are defined by the <td> and </td> tags.. Rows in the header usually use the <th> and </th> tags instead of the <td> and </td> tags. The <th> tag is used to define a header cell, which is usually displayed in bold.
  • Within the individual <td> cells you can put whatever you like: text, images or lists. Your browser will work out the sizes of the cells and rows and columns for you. It is imprtant that you put the same number of cells in each row - if you don't the browser will still display the table but it will look funny.
  • If you do want to change the number of cells in a row you can use the colspan="2" attribute in the <td> tag to make a cell span two or more columns.

Hyperlinks

  • Hyperlinks are defined by the <a> and </a> tags. Everything between these tags will displayed as a clickable link. If it is text it will be underlined and blue, but you can also use images or other HTML elements as links.
  • You must use the href="https://www.example.com" attribute in the <a> tag to specify the URL of the link. You can also use the target="_blank" attribute to open the link in a new tab or window. If you miss out the target="_blank" attribute the link will open in the same tab or window and will replace the page you are on.

Exercise 03

In this exercise you will create a HTML document containing a table and a hyperlink.

You will use the <table>, <thead>, <btody>, <tfoot>, <tr>, <th>, <td>, and <a> tags.

Copy the file template.html and name it exercise-03.html. If you prefer, you can create a new, empty file called exercise-03.html and copy the code from the template file into it.

In the body of the document add a table with a caption and create a hyperlink. Use the exaple at the top of this page as a guide. You can use the same images and text as the example, or you can use your own images and text. You can also use the same table tructure as the example, or you can use your own structure.

When you are finished save your document then in your browser go to http://localhost:8000/exercise-03.html to see what it looks like. Every time you make a change to a file, you need to save it and refresh the page in your browser to see the changes.
