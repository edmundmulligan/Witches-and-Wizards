Welcome to Exercise 07

In this exercise you will create your first Javascript program.

You will create three buttons. The first button will pop up a message when you click it. The second button will turn a message in a <div> on and off when you click it. The third button has no listener so will do nothing when you click it.

What you need to know

  • There are several new concepts in this exercise. First we have a new tag <button> and </button>. Whatever goes between the start and close tags is what is shown on the button. You can even use images insead of text and have a graphical button. Also note that clicking on a button does nothing unless there is some Javascript to tell the browser what to do when you click. The third button does nothing, however many times you click on it.
  • Each button has a new attribute called id="". This is an identifier for the button. You use this id to refer to the button in your Javascript code. You can use whatever you like for the id, but it is a good idea to use something that describes what the button does. And be consistent when you name your buttons - it will make your Javascript code easier to read. The id must be unique in the document, so you can't have two buttons with the same id. You can have buttons with the same id on different web pages, but that is not a good idea. Notice that in our example we included the name of the web page, exercise-07 in the id for each button.
  • Underneath the buttons we have an empty <div> tag with an id of exercise-07-output. This is where the Javascript will write a message when the second button is clicked.
  • We also have our first Javascript code. If some of this explaination sounds a bit like gobledegook, then check out the Advanced Witches and Wizards page for more detailed information about programming jargon. The javascript code is between the <script> and </script> tags.
  • The first three lines of the code in the <script> block defines a function called message(msg). This function takes one argument or parameter, msg, which is the message to be displayed. The function uses the built in Javascript alert() method to display the message in a pop-up window. Javascript has many built in methods and you put them together to make a program.
  • Each tag in the HTML document defines an element. So a <button> tag defines a button element, a <p> tag defines a paragraph element, and so on. Even the <head> and <body> tags define elements, although we never really use the head element. As an aside, when you define styles in CSS, you are defining styles for elements. So when you say h1 { color: red; }, you are saying that all <h1> elements should be red.
  • In the Javascript code we use the document object to access the elements in the HTML document. The document object is a built in object in Javascript that represents the HTML document.It is the root element of the document and contains all the other elements in the document. Javascript can access the elements in the document using the document.getElementById() method. This method takes one argument, the id of the element you want to access. That is why we gave each button an id. Every time you do something in the browser, like moving the mouse, clicking a button, or typing in a text box, the browser sends an event and Javascript can ceate listeners attached to elements which listen for those events and when they hear an event they can do something. So you can use these events to trigger your Javascript code. In this case we are using the addEventListener() method to listen for a click event on a button. When the button is clicked, a function (which depents on which button is clicked) is called and the program does someonething.
  • In our example we have two buttons. The first button has an id of exercise-07-button-1 and the second button has an id of exercise-07-button-2. For each button we use the document.getElementById() method to get the button element. We then use the addEventListener() method to listen for a click event on the button. The addEventListener() method takes two arguments, the event to listen for and the function to call when the event occurs.
  • For the first button, we listen for a click event and call the message() function with the message "Ouch! You clicked my button". This will pop up a message when you click the button.
  • For the second button, we listen for a click event and check if the <div> with the id of exercise-07-output is empty. The innerHTML property of an element is the HTML content of the element. So the innerHTML propert of <div><p>This is a paragraph</p></div> is <p>This is a paragraph</p>. In this case if the <div> is empty, we set the innerHTML property to "You clicked my button too!". If it is not empty, then we can assume we have already clicked the button and written the message so when the listener hears a second click, we set the innerHTML property to an empty string, thus removing the message from the <div>. This is how we can turn the message on and off.
  • Note that even though the third button has an id of exercise-07-button-3, it does not have an event listener, so it does nothing when you click it.

Exercise 07

In this exercise you will create a simple HTML document with some buttons and will write Javascript to do something when you click the buttons. You will use the <button>, and <script> tags, and the id="" attibute.

Copy the file template.html and name it exercise-07.html. If you prefer, you can create a new, empty file called exercise-07.html and copy the code from the template file into it.

Edit the file and add some content. You need to include the <button> and </button> tags, and the <script> and </script> tags and the <div> and </div> tags. Make sure you give each button and the <div> an unique id. You can use whatever you like for the id, but you will need to use the same ids in your Javascript code.

Write some Javascript code to do something when you click the buttons. You can use the code from the example above as a starting point. You can use the document.getElementById() method to get the button elements and the addEventListener() method to listen for a click event on the buttons. You can use the alert() method to display a message when you click the first button. You can use the innerHTML property of the <div> element to display a message when you click the second button.

When you are finished save your document then in your browser go to http://localhost:8000/exercise-07.html to see what it looks like. Every time you make a change to a file, you need to save it and refresh the page in your browser to see the changes.

If you are using Firefox as your browser you can use the Developer Tools to look at the JavaScript console. To do this, right click on the page and select Inspect (Q). This will open a new window at the bottom of the page where you can view the console in the console tab. If you have made any mistakes in your javascript that stop the program from running, you will see an error message in the console. These are known as bugs or syntax errors. There are other kinds of bugs, in particular semantic errors are where the program runs but does not do what you expect it to do. Those are often much harder to find and fix.
