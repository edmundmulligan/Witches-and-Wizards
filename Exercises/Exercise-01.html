<!DOCTYPE html>
<html lang="en">

    <head>
        <title>Witchcraft and Wizardry Exercise 01</title>
    </head>
    <body>
        <header>
            <h1>Witchcraft and Wizardry Exercise 01</h1>
        </header>
        <main>
            <section>
                <h2>Welcome to Exercise 01</h2>
                <p>This is a simple HTML document for exercise purposes.</p>
            </section>
        </main>
        <footer>
            <p>&copy; 2025 Edmund Mulligan</p>
        </footer>
    </body>
</html>

Note the structure of this document.

  • The first line is <!DOCTYPE html> This tells your browser that this is a web page written using the HTML language. All web pages need to start with this line.
  • After the <!DOCTYPE html> line, we have some comments. Comments are not displayed in the browser, but are useful for developers to understand the code. The comments in this document tell you that this is a file for the Witchcraft and Wizardry course and that it is licensed under the MIT License. Comments start with <!-- and end with -->. You can use comments to explain your code, or remind yourself why you wrote something the way you did.
  • After the comments is an <html> tag. This tells your browser that eveything following this line is part of the HTML document. All HTML documents need to start with this and end with a corresponding </html> line. The lang="en" attribute (which comes after the tag name but before the closing >) indicates that the document is in English.
  • HTML describes the structure of a web page. It is not a programming language, but a markup language. HTML uses tags to define elements on the page. Tags are enclosed in angle brackets, e.g., <tagname>content</tagname> and note that (almost) every tag has a corresponding closing tag (that is the same as the opening tag but with a forward slash before the tag name).
  • Tags can also have attributes, which are additional information about the tag. For example, the <html> tag has a lang attribute that indicates the language of the document. Don't forget to look for the closing </html> tag at the end of the document.
  • Every html document has two sections between the <html> and </html> tags: the <head> and <body> sections. The <head> section contains information about the document, such as the title, character set, and links to stylesheets or scripts. The <body> section contains the content of the document that is displayed in the browser.
  • The only tag that *MUST* be in the <head> section is the <title> tag. This tag defines the title of the document, which is displayed in the browser's title bar or tab. The title should be a short and descriptive name for the document. The title is not displayed on the page itself, but it is important for search engines and bookmarks. We will cover other tags that go in the header section in later lessions. As always you must have a closing tag for the title tag and a closing tag for the head section.
  • After the <head> section is the <body> section. This is where the content of the document goes. The content can be text, images, links, and other elements, even videos and sound clips.
  • Evey web page will have different content in the body section but good practice is to have a header section, a main section, and a footer section. If you leve these sections out, the browser will still display the page but it may be harder to read and edit the page later. The header section usually contains the title of the document (this is often the same as the title used in the <head> section but it doesn't have to be) and a navigation menu. The main section contains the main content of the document. The footer section usually contains copyright information and links to other pages. Using structures like this will make it easier for search engines to find your page and for users to navigate your site. They will also make it easier for you to add CSS and JavaScript to your page to make your pages look and work better.
  • The tags in the <body> section to focus on in this exercise are the <h1>, <h2>, and <p> tags.
  • The <h1> and <h2> tags are used for headings. The <h1> tag is used for the main heading of the document and <h2> is used for subheadings. You can have as many <h2> tags as you like, but there should only be one <h1> tag per document. You can also use <h3>, <h4>, <h5>, and <h6> tags for subheadings, but these are not used as often.
  • The <p> tag is used for paragraphs of text. You can have as many <p> tags as you like, but they should be used to break up the content into paragraphs. The <p> tag is one of the very few tags that does not need a closing tag, although it is still good practice to use one.
  • Note that HTML is not case sensitive, so you can use upper or lower case letters for the tags. However, it is good practice to use lower case letters for the tags and attributes. This makes the code easier to read and understand.
  • Also note that HTML ignores extra spaces and line breaks, so you can use them to format your code to make it easier to read. For example, you can put each tag on a separate line, or you can indent the tags to show the structure of the document. This is not required, but it is good practice.
  • We will cover other tags in later lessons as well as look at how to make the page look better using CSS and how to make it work better using JavaScript.
  • Finally, the <footer> section contains copyright information and links to other pages. The copyright information is usually at the bottom of the page and should include the name of the author and the year. The links to other pages can be to other pages on your site or to external sites.
  • The &copy; in the footer is the code for the copyright symbol ©. HTML uses codes like this for displaying special symbols.

Exercise 01

In this exercise you will create a simple HTML document using the tags we have covered in this lesson. You will use the <html>, <head>, <body>, <h1>, <h2>, and <p> tags.

In your editor create a new file called MyExercise-01.html. Edit the file and add the following tags to the document:

  • <!DOCTYPE html>
  • <html lang="en">
  • <head>
  • <title>
  • <body>
  • <h1>
  • <h2>
  • <p>

You can use these extra tags if you like to create a structure for your document, but you can leave them out if you want to.

  • <header>
  • <main>
  • <section>
  • <footer>

Make sure you have a closing tag for each of the tags you add. You can also add some comments using the <!-- and --> tags. You can use the comments to explain what you are doing or to remind yourself what you need to do.

Try to include several <h2> and <p> tags in your document. You can use the <h2> tags for subheadings and the <p> tags for paragraphs of text. You can write about anything you find intersting. If you are stuck for ideas use a heading of "Witches and Wizards" and have subheadings of "Witches" and "Wizards" then write some intersting paragraphs about witches and wizards under each subheading. And if you don't know anything intersting, just make someting up!

When you are finished save your document then open it up in your browser to see what it looks like. If you make any changes to the document, you will need to save the document again and refresh the page in your browser to see the changes.
