What you need to know

  • The first line is <!DOCTYPE html> This tells your browser that this is a web page written using the HTML language. All web pages need to start with this line.
  • In lines 2-5 we have some comments. Comments are not displayed in the browser, but are useful for web witches and wizards to understand the code. The comments in this document tell you that this is a file for the Web Witchcraft and Wizardry course and that it is licensed under the MIT License. Comments start with <!-- and end with -->. You can use comments to explain your code, or remind yourself why you wrote something the way you did.
  • Line 6 is an <html> tag. This tells your browser that eveything following this line is part of the HTML document. All HTML documents need to start with this and end with a corresponding </html> line. The lang="en" attribute (which comes after the tag name but before the closing >) indicates that the document is in English.
  • HTML describes the structure of a web page. It is not a programming language, but a markup language. HTML uses tags to define elements on the page. Tags are enclosed in angle brackets, e.g., <tagname>content</tagname> and note that (almost) every tag has a corresponding closing tag (that is the same as the opening tag but with a forward slash before the tag name).
  • Tags can be nested inside each other. For example, you can have a <p> tag inside a <div> tag. Note you must make sure that you close the tags in the correct order. For example, if you have a <p> tag inside a <div> tag, you must close the </p> tag before you close the </div> tag. If you don't, the browser will not display the page correctly. This means that 
    <div>
    <p>This is a paragraph</p>
</div>
    is valid and will display correctly but 
    <div>
<p>This is a paragraph</div>
</p>
    is not valid and will not display correctly. The browser will try to guess what you mean, but it may not be what you intended. This is called HTML parsing. The browser will try to fix the errors, but it is better to get it right in the first place. If you are not sure if your HTML is valid, you can use a validator. There are many free validators available online. One of the most popular is the W3C Markup Validation Service. This will check your HTML and tell you if there are any errors. It will also give you suggestions on how to fix them.
  • If you are using Visual Studio Code to edit your web pages, the tool will help you by automatically adding the closing tag when you type the opening tag. If you delete the closing tag then when you type </ the editor will automatically add the closing tag for you. This is very helpful, provided you remember this is happening. If you forget and write the whole of the closing tag, you will end up with a bit of HTML nonsense that will show up in the browser and you will need to go back and delete it. So be careful when you are editing your HTML code.
  • Tags can also have attributes, which are additional information about the tag. For example, the <html> tag has a lang attribute that indicates the language of the document. Don't forget to look for the closing </html> tag at the end of the document.
  • Every html document has two sections between the <html> and </html> tags: the <head> and <body> sections. The <head> section contains information about the document, such as the title, character set, and links to stylesheets or scripts. The <body> section contains the content of the document that is displayed in the browser.
  • The only tag that *MUST* be in the <head> section is the <title> tag. This tag defines the title of the document, which is displayed in the browser's title bar or tab. The title should be a short and descriptive name for the document. The title is not displayed on the page itself, but it is important for search engines and bookmarks. We will cover other tags that go in the header section in later lessons. As always you must have a closing tag for the title tag and a closing tag for the head section.
  • <meta> tags in the head section are used to define the information the browser needs to display the page correctly. The meta charset tag on line 9 defines the character set used in the document. The meta viewport tag in line 10 defines how the page should be displayed on different devices. This is important for responsive design, which is a way of making web pages look good on all devices.
  • lines 12-25 are the <body> section. This is where the content of the document goes. The content can be text, images, links, and other elements, even videos and sound clips.
  • Evey web page will have different content in the body section but good practice is to have a header section (N.B. this is not the same as the head section in every HTML document), a main section, and a footer section. If you leave these sections out, the browser will still display the page but it may be harder to read and edit the page later. The header section usually contains the title of the document (this is often the same as the title used in the <head> section but it doesn't have to be) and a navigation menu. The main section contains the main content of the document. The footer section usually contains copyright information and links to other pages. Using structures like this will make it easier for search engines to find your page and for users to navigate your site. They will also make it easier for you to add CSS and JavaScript to your page to make your pages look and work better.
  • The tags in the <body> section to focus on in this exercise are the <h1>, <h2>, and <p> tags.
  • The <h1> tag in line 14 and <h2> tag in line 18 are used for headings. The <h1> tag is used for the main heading of the document and <h2> is used for subheadings. You can have as many <h2> tags as you like, but there should only be one <h1> tag per document. You can also use <h3>, <h4>, <h5>, and <h6> tags for subheadings, but these are not used as often.
  • The <p> tag in lines 19 and 23 is used for paragraphs of text. You can have as many <p> tags as you like, but they should be used to break up the content into paragraphs. The <p> tag is one of the very few tags that does not need a closing tag, although it is still good practice to use one.
  • Note that HTML is not case sensitive, so you can use upper or lower case letters for the tags. However, it is good practice to use lower case letters for the tags and attributes. This makes the code easier to read and understand.
  • Also note that HTML ignores extra spaces and line breaks, so you can use them to format your code to make it easier to read. For example, you can put each tag on a separate line, or you can indent the tags to show the structure of the document. This is not required, but it is good practice.
  • We will cover other tags in later lessons as well as look at how to make the page look better using CSS and how to make it work better using JavaScript.
  • Finally, the <footer> section in lines 22-24 contains copyright information and links to other pages. The copyright information is usually at the bottom of the page and should include the name of the author and the year. The links to other pages can be to other pages on your site or to external sites.
  • The &copy; in line 23 is the code for the copyright symbol ©. HTML uses codes like this for displaying special symbols.

Copy the file template.html and name it exercise-01.html. If you prefer, you can create a new, empty file called exercise-01.html and copy the code from the template file into it.

Advanced Witches and Wizards

When you create a web page you create a document. The tags that you use define the structure of your documrnt and in later exercises you will learn how to use that structure, knows as the Document Object Model or DOM, to change tha appearance of sections of your web page with styles or to interact with the user of you web page using Javascript.

The DOM is a hierarchical or tree structure. This means that the tags are arranged in a tree-like structure - imagine a tree with the trunk in the ground (ignore the roots as this metaphor only works so far) and the main branches coming out of the trunk then smaller branches coming out of the main branches and so on. The <html> tag is the trunk of the tree, and the <head> and <body> tags are the main branches. The other tags are the smaller branches and leaves. The tree structure is important because it allows you to select and manipulate elements in the document using CSS and JavaScript.

Imagine you have a web page that looks like this: 

<html>
    <head>
        <title>My Web Page</title>
    </head>
    <body>
        <h1>Welcome to my web page</h1>
        <section>
            <h2>About me</h2>
            <p>My name is Edmund Mulligan and I am a web developer.</p>
            <h2>My hobbies</h2>
            <p>I am a STEM ambassador, I sing in a Male Voice Choir and I teach Nonviolent Communication.</p>
        </section>
     </body>
</html>

The DOM model of this document would look like this:

  • html
    • head
      • title: "My Web Page"
    • body
      • h1: "Welcome to my web page"
      • section
        • h2: "About me"
        • p: "My name is Edmund Mulligan and I am a web developer."
        • h2: "My hobbies"
        • p: "I am a STEM ambassador, I sing in a Male Voice Choir and I teach Nonviolent Communication."

Note that the elements in the DOM match the tags in the HTML document, but there are no closing tags in the DOM. You can reference the elements of the DOM using the dot notation. For example, you can reference the <h1> element using document.body.h1. You can also reference the <h2> elements using document.body.section.h2. In later exercises you will learn how to give elements ids and classes so you can distinguish between elements of the same type.
